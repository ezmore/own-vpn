<b>项目需求说明</b><br>
一、 核心背景与资源条件
拥有两台跨域的计算资源，希望打破地域限制将其打通：

公网服务端 (VPS)：拥有固定公网 IP，上/下行带宽为 200Mbps，系统为 Ubuntu 22.04。

内网客户端 (局域网内服务器)：带宽为 100Mbps，系统需要兼容 Ubuntu 22.04 和 Windows 10 两种环境（即兼容Linux和Windows环境）。

二、 最终核心功能需求归纳
制作一套虚拟局域网（SDN/VPN）互联系统，并具备以下四个维度的硬性指标：

1. 极简的网络架构与大网段支持
A 类大网段：组网后虚拟局域网的 IP 分配必须为 A 类私有网段（即 10.0.0.0/8），能够容纳海量互联节点。

极简镜像体积：为了追求极致的文件大小与轻量化，Docker 容器内部环境基于最新的 Alpine Linux 系统进行构建，确保无冗余垃圾组件。

2. 可视化 Web 管理后台
服务端容器需要内置一个 Web 管理页面，且固定在 41090 端口。

该后台用于动态生成客户端所需的组网认证信息/明文配置文件，以便客户端能秒级接入。

3. 精细化的网络行为控制 (服务端 Web 控制面板实现)
在 Web 服务端的 41090 端口页面上，能够针对每一个连接的客户端节点单独设置以下三大限制规则：

开放服务的连接端口：可动态设置该节点允许/拒绝访问的特定端口白名单。

连接速度上限设置：根据宿主机带宽上限，动态限制单个客户端的最高传输速率。

连接数量上限设置：动态限制单个客户端发起的最大网络并发连接数，防止流量异常。

4. 标准化的 Docker 镜像与现代化部署方式
一键式部署：所有 Docker 容器的生成、编译、环境变量配置和启动，必须全部采用标准的 docker-compose.yml (YAML 形式) 进行统一编排与拉起。

双系统客户端方案，客户端必须提供 Ubuntu 22.04 容器方案 和 Windows 10 原生/无隔离接入方案，确保全场景下的网络互通与低延迟。

三、 总结
一套基于 Alpine 基础镜像、采用 docker-compose 编排、以 WireGuard + 动态内核沙箱（tc/iptables）为底层、自带 41090 Web 控制台的轻量化跨国虚拟局域网（A类网）一站式构建与全平台部署方案。
